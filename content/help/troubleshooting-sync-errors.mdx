# Troubleshooting Sync Errors

If you're experiencing sync errors, this guide will help you identify and resolve common issues.

## Common Error Types

### 1. Authentication Failures

**Error Message**: `Authentication failed: Invalid credentials`

**Causes**:
- Expired API tokens
- Changed passwords
- Revoked permissions

**Solution**:
1. Go to **Integrations** → **[Your Integration]**
2. Click **Reauthorize**
3. Enter updated credentials
4. Test the connection

### 2. Rate Limit Errors

**Error Message**: `Rate limit exceeded. Retry after X seconds`

**Causes**:
- Too many API calls in a short period
- Aggressive sync frequency settings

**Solution**:
- Reduce sync frequency in settings
- Enable rate limit handling (automatic retry)
- Consider upgrading your API tier

### 3. Data Validation Errors

**Error Message**: `Validation failed: [Field] is required`

**Common Fields**:
- Missing SKU
- Invalid price format
- Blank product names

**Solution**:
```javascript
// Example: Add validation rules
const validationRules = {
  sku: { required: true, pattern: /^[A-Z0-9-]+$/ },
  price: { required: true, min: 0 },
  name: { required: true, maxLength: 255 }
}
```

## Debugging Steps

### Step 1: Check Sync Logs

1. Navigate to **Monitoring** → **Sync Logs**
2. Filter by:
   - Date range
   - Integration
   - Error type
3. Click on failed syncs for details

### Step 2: Verify Data Mapping

Ensure fields are correctly mapped:

- **product_id** → **sku**: ✅ Mapped
- **list_price** → **price**: ✅ Mapped  
- **qty_available** → **quantity**: ❌ Not mapped

### Step 3: Test Individual Records

Test sync with a single record:

```bash
# Test single product sync
truthsource sync --type product --id PROD-123 --debug
```

## Specific Error Solutions

### "Duplicate Key" Errors

**Problem**: Trying to create a record that already exists

**Solutions**:
1. Enable upsert mode (update if exists)
2. Add unique identifier mapping
3. Run deduplication check

### "Field Type Mismatch" Errors

**Problem**: Data type doesn't match expected format

**Example**:
```json
// Error: Expected number, got string
{
  "price": "$99.99" // Should be 99.99
}
```

**Fix**: Configure data transformations in mapping

### "Missing Parent Record" Errors

**Problem**: Trying to sync child record before parent

**Solution**: Enable dependency resolution:
- Sync products before inventory
- Sync customers before orders
- Use cascade sync option

## Prevention Best Practices

### 1. Set Up Monitoring

- Enable email alerts for sync failures
- Set threshold alerts (e.g., >5% failure rate)
- Create Slack notifications for critical errors

### 2. Implement Retry Logic

```javascript
const retryConfig = {
  maxAttempts: 3,
  backoffMultiplier: 2,
  initialDelay: 1000 // 1 second
}
```

### 3. Regular Health Checks

- Schedule weekly sync audits
- Review error patterns
- Update field mappings as needed
